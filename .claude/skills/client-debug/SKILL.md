# Client Debug

## Phase 1: 증상 파악

인자에서 다음을 확인한다:
- 어떤 상황에서 발생하는가? (재현 조건)
- 기대한 동작과 실제 동작의 차이는?
- 에러 메시지나 로그가 있는가?

정보가 부족하면 질문한다.

---

## Phase 2: 원인 탐색

재현 조건을 바탕으로 관련 파일을 Grep/Glob으로 탐색한다:
- 증상과 관련된 시스템 파일 찾기
- 버그가 발생하는 코드 경로 특정
- 의도치 않은 상태 변화나 signal 누락 확인

---

## Phase 3: 수정

원인이 특정되면 수정한다.
수정 범위는 버그 원인에 한정한다 — 관련 없는 코드 리팩토링 금지.

---

## Phase 4: 요약 출력

```
## 버그 수정: [증상 한 줄 요약]

**원인**: [원인 설명]
**수정 파일**: `[경로]` — [변경 내용 한 줄]
**재현 조건에서 검증**: [수동 확인 필요 항목]

다음 단계: /client-review [수정된 파일경로]
```
