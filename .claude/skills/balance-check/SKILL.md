## Phase 1: 분석 도메인 파악

`$ARGUMENTS[0]`에서 도메인 결정:

- **economy** → 재화 지급량, 카드 가격, 코스트 한도, 업그레이드 비용
- **combat** → 유닛 스탯 (체력/공격/속도), 죄종 시너지 수치, 직업 시너지 수치
- **cards** → 카드 코스트 분포, 등급별 강도, 상점 출현율
- **파일 경로 입력** → 해당 파일 직접 로드 후 도메인 추론

인자 없으면 어떤 시스템을 확인할지 질문한다.

---

## Phase 2: 데이터 파일 읽기

`src/data/` 및 `docs/game_design/` 에서 관련 파일을 읽는다.
`docs/game_design/GAME_DESIGN.md`를 읽어 설계 의도와 기댓값 범위를 파악한다.

---

## Phase 3: 분석 실행

**경제 밸런스:**
- 라운드당 재화 지급량 vs 카드 평균 가격 비율
- 코스트 한도 업그레이드 비용 곡선
- 리롤 비용 대비 기대값
- 재화 이월이 후반 런에 미치는 영향

**전투 밸런스:**
- 직업별 DPS/생존력 비교
- 죄종 시너지 브레이크포인트 수치 검토
- 저코스트 다수 vs 고코스트 소수 편성 효율 비교
- 출전 코스트 한도 대비 최적 편성 존재 여부

**카드 밸런스:**
- 코스트 대비 전투 기여도
- 특정 카드/죄종/직업 조합이 항상 최선인지 (지배 전략)
- 업그레이드 분기 간 밸런스
