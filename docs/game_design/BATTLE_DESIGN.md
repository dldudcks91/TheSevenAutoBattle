# TheSevenAutoBattle — 전투 설계 문서

> 최종 업데이트: 2026-05-01

---

## 1. 전투 철학

- 플레이어는 **SHOP 단계까지만** 결정한다. 전투가 시작되면 개입 없음.
- 결과는 오직 편성·아이템·배치 선택의 결과물이어야 한다.
- 전투는 **완전 정보 기반** — 적 편성은 SHOP 단계에 공개된다.

---

## 2. 전투 진행 구조

```
[전투 시작]
    ↓
플레이어 유닛·적 유닛 동시 스폰
    ↓
매 프레임: 모든 유닛 AI 동시 실행 (오토배틀)
    ↓
어느 쪽이 전멸하면 전투 종료
    ↓
[결과 처리] 승리 시 보상 지급 → 다음 라운드 / 패배 시 런 종료
```

전투 중 플레이어 입력은 **유닛 클릭 → 정보 확인**뿐. 전투에 영향을 주지 않는다.

---

## 3. 유닛 AI

### 상태 머신

각 유닛은 매 프레임 아래 상태 중 하나에 있다.

```
IDLE ──→ MOVE ──→ ATTACK
  ↑                  │
  └──────────────────┘
         (쿨타임)

ATTACK / MOVE / IDLE ──→ HURT (피격 시 플래시)
HURT ──→ DEAD
```

### 매 프레임 판단 순서

1. **타겟 유효성 검사** — 타겟이 없거나 죽었으면 가장 가까운 적으로 재탐색
2. **거리 측정** — 타겟까지 거리 계산
3. **행동 결정**
   - 사거리 밖 → MOVE (타겟 방향으로 이동)
   - 사거리 안 + 쿨타임 만료 → ATTACK 시작
   - 사거리 안 + 쿨타임 대기 중 → IDLE 대기
4. **분리 적용** — 아군 유닛과 겹침 보정 (항상)

### 타겟 선택 규칙

- **가장 가까운 적** 단일 타겟
- 타겟이 죽으면 즉시 다음 가장 가까운 적으로 전환
- 특별한 우선순위 없음 (거리 기반 단순 탐색)

> 향후 확장 여지: 직업별 우선 타겟 (e.g., 힐러가 체력 낮은 아군 우선 타겟), 도발 메카닉 등

---

## 4. 공격 시스템

### 공격 흐름

```
[공격 시작]
    ↓
애니메이션 재생
    ↓
애니메이션 일정 지점 도달 → 피해 적용  ← hit_frame_ratio
    ↓
애니메이션 종료 → 쿨타임(attack_speed) 시작
    ↓
쿨타임 만료 → 다시 공격 가능
```

피해 적용 타이밍(`hit_frame_ratio`)은 유닛마다 다르게 설정할 수 있으며, 애니메이션과 타격감이 일치하도록 조정한다. 구체값은 `src/data/units/units.csv` 참고.

### 근접 vs 원거리

| | 근접 (is_ranged: false) | 원거리 (is_ranged: true) |
|---|---|---|
| 피해 방식 | 타겟에 직접 take_damage() | 발사체 생성 → 도달 시 피해 |
| 유효 거리 체크 | 피해 적용 시점에 재확인 | 발사체가 독립적으로 추적 |

---

## 5. 피해 계산

```
실제 피해 = max(최소 피해, 공격력 - 방어력)
```

- 방어력이 아무리 높아도 최소 피해는 항상 들어간다 — 방어력으로 완전 무효화 불가
- 최소 피해 / 방어력 수치는 데이터 파일 참고

---

## 6. 발사체 시스템

원거리 유닛 전용.

- 발사체는 **호밍(Homing)** — 타겟이 살아있는 동안 방향을 지속 갱신
- 타겟이 죽으면 마지막 방향으로 직진 후 소멸 (빗나감 연출)
- 발사체 속도·충돌 반경은 데이터 파일 참고

---

## 7. 충돌 처리

### 분리(Separation)

- **같은 팀** 유닛끼리만 적용
- 두 유닛의 몸체 반경 합보다 가까우면 서로 밀어냄
- 공격 중에도 항상 적용 → 뭉치는 현상 방지

### 전면 차단(Front Block)

- 이동 방향 정면에 **적** 유닛이 있으면 X축 이동을 차단
- Y축 이동은 허용 → 자연스럽게 옆으로 비켜가는 흐름 형성
- 효과: 일렬 종대로 좁은 공간을 통과하는 대신 측면으로 우회하는 움직임

---

## 8. 배치 시스템

### 그리드 → 전장 매핑

- 전투 시작 시 점유된 셀이 플레이어 **좌측 구역**에 유닛을 자동 스폰
- 셀의 3×3 좌표가 초기 스폰 위치를 결정 — 자유 드래그 배치 없음
- 셀 내 수량 N → N개 유닛을 셀 중심 주변에 **스태거 배치** (겹침 방지)
- 셀 위치는 초기 이동 방향·충돌 흐름에 영향 → 전면/후방 배치 의사결정 요소
- 적은 고정 위치에 순서대로 스폰 (편성 공개되므로 카운터 배치 가능)
- **SHOP ↔ Battle 좌표 일치:** SHOP PlayerZone/EnemyZone 토큰의 글로벌 좌표를 그대로 전투 스폰 좌표로 사용. 별도의 BATTLE 좌표계로 확장/리매핑하지 않는다 → 사용자가 SHOP에서 본 위치 그대로 전투가 시작된다.
- **토큰·유닛 시각 일치:** 핸드 카드 초상화 / 드래그 프리뷰 / prep 필드 토큰 / 전투 유닛 sprite 가 모두 `unit_data.sprite_scale * 0.75` 로 동일 스케일.

> 향후 지형·엄폐물 도입 시 셀 위치의 전략적 가중치 상승 가능.

### 전투 스탯 적용 순서

전투 시작 시 각 유닛의 최종 스탯은 아래 순서로 합산된다.

```
1. 기본 스탯          (units.csv)
2. 레벨 보너스        (셀 레벨 기반)
3. 스킬 브랜치 보너스  (선택한 분기)
4. 아이템·유물 효과   (EffectiveStats.from_slot)
```

> 현재 `EffectiveStats.from_slot(slot)`은 아이템 효과만 처리한다. 레벨·브랜치 입력을 받도록 확장 필요 (구현 시 주의).
