# 아이템 시스템 — 기획 문서

> 최종 업데이트: 2026-05-01
> 상태: 구조 개편안 (유물형)
> 대응 개발 계획: [ITEM_DEV_PLAN.md](ITEM_DEV_PLAN.md)

---

## 1. 컨셉

- **카드 = 누구를 부르냐**, **아이템 = 플레이어가 들고 가는 유물.**
- 아이템은 유닛이 아니라 **플레이어가 소유**한다. 한 번 얻으면 모든 라운드에 영향을 준다.
- 효과 적용 범위는 아이템마다 다르다 — 일부는 아군 전체에, 일부는 조건을 만족하는 유닛에만 적용.
- [GAME_DESIGN.md](GAME_DESIGN.md) §1의 정보 공개 원칙을 그대로 계승: 모든 아이템 효과는 상점/인벤토리에서 수치까지 노출. 숨겨진 효과 없음.
- "정보는 다 줄게. 그걸로 얼마나 아끼냐가 실력이야." — 아이템도 동일.

---

## 2. 슬롯 구조

- 플레이어는 **유물 슬롯을 고정 개수만큼** 보유 (구체 칸 수는 `src/data/balance.csv`).
- 유닛별 슬롯은 없다. 아이템은 카드/유닛에 장착하는 개념이 아니라 **플레이어 인벤토리에 누적**된다.
- 슬롯이 가득 찬 상태에서 추가 구매 시: 기존 유물 1개를 버려야 획득 가능 (교체).
- 같은 유물 중복 보유 가능 여부는 미결 (§7).
- 유물의 효과는 전투 시작 시 자동 발동 — 장착·해제 행위 자체가 없다.

---

## 3. 효과 적용 범위 (Scope)

각 아이템은 **scope 필드**를 가진다. 데이터 단의 분기 키이며, 효과 풀의 다양성은 이 필드로 만든다.

| Scope | 의미 | 예시 컨셉 |
|-------|------|----------|
| `ALL_ALLIES` | 아군 유닛 전부 | 모든 아군 능력치 상향 |
| `CONDITIONAL` | 조건을 만족하는 아군만 | 특정 카테고리/태그 유닛만 상향, 첫 소환 유닛만 상향 등 |
| `GLOBAL` | 유닛이 아닌 메타 자원에 작용 | 라운드 보상·리롤·고용 비용 등 (후속 확장 여지) |

조건부(`CONDITIONAL`)의 구체 조건 종류와 글로벌(`GLOBAL`) 효과 종류는 후속 단계에서 채운다 (§7).

---

## 4. 효과 종류 (보류)

구체 옵션 풀(스탯 상향/경제 보너스/트리거 등)은 **이번 구조 개편 범위 외**. 기존에 검토하던 4종 스탯(이속·공격·체력·방어)을 유물 효과로 전환할지, 글로벌 경제 유물을 추가할지는 §1 방향 확정 후 별도 결정.

이 시점에 확정된 것:
- 효과 데이터는 [src/data/items/items.csv](../src/data/items/items.csv) 로 관리. 표시명은 `name_key` 컬럼에 번역 키만 두고, 실제 한글/영문 라벨은 [src/data/i18n/text.csv](../src/data/i18n/text.csv) 에서 관리한다.
- 효과 적용 시점은 **전투 시작 시 1회 합산** — 전투 중 동적 트리거 없음 (MVP 가정).

---

## 5. 획득 UX

### 획득
- 매 라운드 SHOP 핸드(랜덤 5장)에 **ITEM 카드**가 다른 카드 종류와 섞여 등장한다. 가중치는 [GAME_DESIGN.md §4](GAME_DESIGN.md) 참고.
- 카드를 클릭하면 골드 차감 + 플레이어 인벤토리에 추가 (현재는 더미 — 인벤토리 추가 미구현).
- 리롤(핸드 좌측 버튼, `reroll_cost` g)은 카드/유물 구분 없이 핸드 전체를 다시 뽑는다.

### 인벤토리
- 상점 화면에 **플레이어 유물 패널** 표시 (고정 슬롯 그리드).
- 슬롯이 가득 찬 상태에서 신규 구매 → 버릴 유물 선택 모달 → 교체 확정.
- 빈 슬롯이 있으면 자동 추가.

### 정보 공개
- 적은 유물을 보유하지 않음 (MVP).
- 각 유물 카드: `이름 + scope 표시(전체/조건/글로벌) + 효과 텍스트 + 가격`.
- 조건부 유물은 어떤 조건인지 텍스트로 명확히 표기 (예: "근접 유닛에게").

---

## 6. 가격

- 유물 가격은 `src/data/items.csv` 행마다 기재 (실효 도입 시).
- 더미 단계 임시 가격은 `RosterSlot.dummy_price` 에 5g 고정 (`run_state.gd` 의 `_roll_hand_offer`).
- `src/data/balance.csv`의 라운드 기본 보상과 카드 고용 비용 사이에서, "병사 고용 + 가끔 유물 1개" 동선이 자연스럽도록 설계.
- 슬롯이 차서 교체할 때의 환불 여부는 미결 (§7).

---

## 7. MVP 범위

**포함:**
- 플레이어 인벤토리 (고정 슬롯).
- 유물 데이터 구조: `id`, `name_key`(번역 키 → i18n CSV 참조), `scope`, `condition`(scope=CONDITIONAL일 때), `effect_payload`, `price`.
- 상점 유물 오퍼 / 구매 / 교체 흐름.
- 전투 시작 시 효과 합산 적용 파이프라인 (효과 종류는 별도 PR에서 채움).

**제외 (후속):**
- 구체 효과 풀 (스탯/경제/트리거).
- 등급·세트 효과·유물 진화.
- 적/보스 유물 보유.
- 영속 인벤토리 (런 종료 후 보관).
- 트리거형 효과 (피격 시·처치 시).

---

## 8. 후속 확장 메모

- **등급 도입:** 유물 데이터에 `tier` 필드 추가 + 가격 곡선.
- **세트 효과:** 특정 유물 조합 시 보너스 — 향후 설계.
- **트리거 효과:** scope에 `TRIGGER` 추가하고 이벤트 키 정의.
- **글로벌 경제 유물:** 라운드 보상·리롤 비용·고용 비용 수정자.
