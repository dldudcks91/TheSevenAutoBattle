# TheSevenAutoBattle — 게임 디자인 문서

> 최종 업데이트: 2026-05-02

## 1. 게임 개요

- **장르:** 덱빌딩 로그라이크 + 오토배틀 (PvE)
- **플랫폼:** Steam (PC)
- **핵심 한 줄:** "정보는 다 줄게. 그걸로 얼마나 아끼냐가 실력이야."
- **수익모델:** 스팀 단일 구매 (인앱결제 없음)

## 2. 핵심 루프

각 라운드는 **3단계**로 진행되며, 각 단계는 별도의 화면(씬)으로 분리된다. 한 단계의 의사결정이 다음 단계의 입력이 되는 구조다.

> **자산 모델**: 플레이어는 **3×3 그리드로 이루어진 영구 군대**를 소유한다. 셀에 병사를 배치하면 런이 끝날 때까지 유지된다. 영구 자산은 골드, 그리드(배치된 병사), 장착된 스킬·아이템 카드다.
> 골드 사용처: **카드 구매** (영웅·스킬·아이템 카드 중 선택).

```
[런 시작] 골드 보유, 그리드 비어 있음, 핸드 5장 추첨

[매 라운드]
  1. SHOP 단계
     - 우측: 적 편성 공개
     - 핸드: 랜덤 5장(영웅 / 강화 / 스킬 / 아이템 중 무작위)
     - 영웅 카드를 셀에 드래그 — 셀당 한 종류, 같은 영웅 카드를 더 올리면 누적
     - 비-영웅 카드(강화·스킬·아이템)는 클릭 시 즉시 골드 차감 + 카드 소멸
     - 핸드 좌측 리롤(−3 g): 카드만 새로 뽑힘, 그리드의 병사는 보존됨
     - 한번 사용한 카드는 핸드에서 사라진다 (셀 우클릭 회수는 같은 라운드 내 리롤 전에만 핸드로 복귀)
     - "전투 시작" 으로 출전 확정 — 그리드 병사 비용 일괄 차감

  2. 전투 단계
     - 오토배틀 진행, 플레이어 개입 없음
     - 전투 후 그리드는 보존 — 다음 라운드 SHOP 진입 시 그대로 유지, 사망자 포함 모두 풀 HP로 부활

  3. 전투 종료
     - 결과·간단 통계 확인 (승/패, 처치 수, 획득 재화 등)
     - 다음 라운드 진입 시 핸드 5장 새로 추첨
     - 다음 라운드 진입 또는 런 종료

[런 종료] 패배 or 보스 클리어 → 모든 자산 초기화
```

**왜 이 모델인가:**
- 그리드 배치는 런 전체에 걸친 **누적 투자** — 어느 셀에 무엇을 올릴지는 런 전반의 방향을 결정
- 카드로 영웅·스킬·아이템을 구매 → 골드 긴장감은 카드 선택과 구매 타이밍

**왜 3단계인가:**
- SHOP 단계는 **전술 의사결정** (적 정보 공개 → 카드 구매로 즉각 대응할지 결정)
- 3단계는 라운드 간 호흡과 피드백 (이번 결정이 잘 됐는지 확인)

**핵심 긴장감:**
- 적 구성 확인 후 어떤 카드를 살지 결정 → 정보 기반 투자 딜레마
- 영웅·스킬·아이템 카드 사이의 자원 딜레마 (즉시 전력 vs 장기 투자)
- 그리드가 쌓일수록 강해지므로, 이른 투자와 늦은 투자 간의 타이밍 긴장감

**런 길이 — 10웨이브:**
- 한 런은 총 10웨이브로 구성.
- W10이 최종 보스(현재 werebear 중심). 클리어 시 RUN CLEAR.

> 각 단계 화면의 입출력·전환 계약은 `docs/game_design/SCENES.md` 참고.

## 3. 병사(유닛) 시스템

### 기본 규칙
- **병사는 영구 소유 자산이다.** 구매한 영웅 카드는 그리드에 등록되며 런 전체에서 유지된다
- 셀 하나에는 **한 종류**의 병사만 배치 가능
- **셀당 최대 4명** — 셀 내부는 2×2 사분면(좌상/우상/좌하/우하)으로 분할되어 한 칸에 한 병사씩 점유한다. 5번째 영웅 카드 배치는 거부된다(다른 셀로 분산해야 함).
- 같은 종류의 병사를 **서로 다른 셀**에 배치할 수 있음 — 각 셀은 독립된 컨텍스트를 가짐
- 셀 안의 병사들은 전투 시 각각 **개별 유닛**으로 출전 (SHOP에서 본 사분면 위치가 그대로 출전 좌표)
- 부착된 스킬·아이템 카드는 해당 셀의 병사 전체에 동일 적용

### 에셋 (Tiny RPG Character Asset Pack v1.03)
아군 유닛 9종, 적 몬스터 11종 사용.

## 4. 카드 시스템

매 라운드 랜덤 풀에서 **핸드 5장**(`HAND_OFFER_COUNT`, `balance.csv` 의 `hand_offer_count` 키로 조정)이 추첨된다. 카드는 4종류이며 가중치 랜덤으로 섞여 등장한다.

| 카드 종류 | 가중치 | 동작 |
|---|---|---|
| **영웅 카드 (HERO)** | 60% | 그리드 셀에 드래그해 배치. 셀당 한 종류만(같은 영웅은 누적). 비용은 "전투 시작" 시 일괄 차감. |
| **강화 카드 (UPGRADE)** | 14% | 스탯 지정 강화 — **공격력 강화(+10 ATK)** / **체력 강화(+50 HP)** / **방어력 강화(+2 ARM)** 중 하나가 랜덤 등장. 드래그로 셀에 적용, 해당 셀 유닛 전체에 영구 적용. 셀당 최대 +3회(★ 표시). **그리드에 유닛이 있을 때만 등장**. |
| **스킬 카드 (SKILL)** | 13% | **같은 핸드에 등장한 영웅 카드의 스킬만 등장** — 영웅 카드가 없으면 HERO로 귀속. 카드명은 스킬명 그대로 표시. 드래그로 해당 영웅이 있는 셀에 적용. |
| **아이템 카드 (ITEM)** | 13% | 아이템 효과. 드래그로 셀에 적용. |

**핵심 규칙**
- **1회용:** 한 번 셀에 올리거나 클릭으로 소모한 카드는 핸드에서 사라진다.
- **회수:** 그리드 셀 우클릭으로 카드를 빼낼 수 있다 — 같은 라운드 내 리롤 전이라면 핸드로 복귀, 그 외에는 폐기.
- **리롤 (`reroll_cost` g):** 핸드 좌측 버튼. 카드만 다시 뽑는다 — **그리드에 배치된 병사는 그대로 유지**된다.
- **셀 규칙:** 셀 하나에는 한 종류의 영웅만 배치 가능. 같은 영웅 카드는 최대 **4명**까지 누적되며, 셀 내부 2×2 사분면에 분산 표시된다. 5번째 시도는 거부된다.
- 동일 카드 풀의 중복 등장은 허용 (with replacement).

데이터: 가중치·풀 구성은 `src/economy/run_state.gd` 의 `_roll_hand_offer()` 에서 관리. 더미 가격은 RosterSlot.dummy_price 필드.

## 6. 밸런스 데이터 관리

경제 수치(골드·보상·카드 가격 등)는 코드가 아닌 CSV에서 관리 — 코드 수정 없이 조정 가능.

| 파일 | 내용 |
|------|------|
| [src/data/balance.csv](../../src/data/balance.csv) | 경제 상수 (시작 골드, 라운드 보상, 리롤 비용 등) |
| [src/data/rounds.csv](../../src/data/rounds.csv) | 웨이브별 적 라인업 |
| [src/data/units/units.csv](../../src/data/units/units.csv) | 유닛 기본 스탯 (아군/적 구분 포함). 표시명은 `name_key` 컬럼에 번역 키만 둔다. |
| [src/data/items/items.csv](../../src/data/items/items.csv) | 아이템 옵션·가격. 표시명은 `name_key` 컬럼에 번역 키만 둔다. |
| [src/data/i18n/text.csv](../../src/data/i18n/text.csv) | 다국어 텍스트 (`keys,ko,en`) — Godot이 자동으로 `text.ko.translation` / `text.en.translation` 생성. 코드에서 `tr(key)`로 사용. |

> 데이터 CSV는 모두 `.import` 가 `importer="keep"` 으로 설정돼 있어 Godot의 csv_translation 임포터를 거치지 않는다 (컬럼명이 로케일로 오인되면 안 되므로). i18n CSV(`src/data/i18n/`)만 csv_translation으로 자동 임포트된다.

## 7. 미결 항목

| 항목 | 상태 |
|------|------|
| 카드 공개 수 및 리롤 비용 | 결정 — 5장 / `reroll_cost=3` ([src/data/balance.csv](../../src/data/balance.csv)). 플레이테스트 후 조정. |
| 카드 풀 가중치 (HERO 60% / UPGRADE 14% / SKILL 13% / ITEM 13%) | 잠정 — 더미 단계 가중치. 실효 카드 도입 후 재조정. |
| 영웅 카드 슬롯 수 (셀당 최대 영웅 수) | 결정 — 셀당 최대 4명. 셀 내부 2×2 사분면에 한 병사씩 점유. 5번째 배치는 거부. |
| 스킬 카드 슬롯 수 (영웅당 최대 스킬 수) | 결정 — 기본 1 + 카드 슬롯 2 = 최대 3개 |
| UPGRADE / SKILL / ITEM 카드 실효 효과 | 미결 — 현재 클릭 시 골드만 차감하는 더미 |
| 셀 초기화/타입 변경 비용 또는 불가 여부 | 미결 |
| 라운드 간 그리드 영속화 (영구 자산) | 결정 — 그리드는 런 전체 유지. 사망자도 풀 HP로 부활. 비용은 신규 배치분만 차감. 회수는 unpaid entry에만 가능 (paid는 영구 자산). |
| 셀 비우기/판매 (paid entry 처분) | 미결 — 현재 슬라이스는 추가 + 셀 간 스왑만 허용. 환불·셀 비우기는 후속. |
| 사망 페널티 | 결정 — 없음 (전원 풀체력 부활). 향후 변경 시 재논의. |
| 영웅 카드 → `RunState.roster` 영구 등록 동선 | 미결 — 현재는 `RunState.grid_cells`(셀 단위)만 영구 보존. 평면 roster 인덱스가 필요한 시점에 결정. |
| 보스 메카닉 | 결정 — W10 최종 보스(werebear). 호위로 elite_orc(전체 강화 → 보스 데미지 증폭) + werewolf 채택. 보스 고유 스킬은 미결. |
| 웨이브별 적 라인업 | 결정 — 챕터별 학습 곡선 잡힘 ([WAVE_DESIGN.md](WAVE_DESIGN.md), [src/data/rounds.csv](../../src/data/rounds.csv)). 플레이테스트 1회 후 카운트 조정. |
| 웨이브별 골드 보상 곡선 | 결정 — `reward_per_round + reward_growth_per_round × wave_index` ([src/data/balance.csv](../../src/data/balance.csv)). 플레이테스트 후 수치 조정. |
| 패배 페널티 구조 | 미결 |
